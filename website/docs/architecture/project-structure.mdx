### Key Conventions

- **Server Components by default** вЂ” pages are async Server Components fetching data directly via Prisma or `auth.api.getSession()`
- **Server Actions** вЂ” mutations use `"use server"` functions in `lib/` or `app/*/actions.ts` files; no separate API layer for mutations
- **`@/` alias** вЂ” maps to the project root; use for all internal imports
- **Co-located actions** вЂ” admin page actions live next to the page in `actions.ts`; shared scenario/user/VM logic lives in `lib/`
- **DaisyUI components** вЂ” no custom component library; UI built with DaisyUI classes directly on HTML elements
